# CampusPulse — Complete Product & Engineering Master Prompt

## Role

You are a Senior Software Architect, Product Designer, Security Engineer, and Mobile App Lead.

Your task is to design the complete architecture, database, APIs, security model, backend logic, and mobile application structure for a college-exclusive social platform called **CampusPulse**.

The system must be production-ready, scalable, secure, and designed for thousands of concurrent users.

---

# Product Vision

CampusPulse is a hybrid college communication platform that combines:

* Verified student networking
* College communities
* Study groups
* Direct messaging
* Event discovery
* Anonymous confessions
* Anonymous discussion boards
* Blind chat matching

The platform has two completely separate identities:

### Verified Identity

Used for:

* Profile
* Networking
* Study groups
* Public posts
* Direct messages
* Clubs and communities

### Anonymous Identity

Used for:

* Confessions
* Anonymous discussions
* Blind chat
* Anonymous comments

No ordinary user, administrator, database leak, or API response should be capable of connecting anonymous activity to a student's real identity.

# Security Architecture

Anonymous system must be completely isolated.

## Anonymous Identity Layer

Never connect:

Anonymous Post → User ID

directly.

Instead:

User
→ Anonymous Session Token
→ Anonymous Alias
→ Anonymous Activity

Use rotating mappings.

Database leaks must not reveal identities.

---

# Moderation System

## AI Moderation

Before database write:

Scan for:

* Hate speech
* Harassment
* Threats
* Doxxing attempts
* Phone numbers
* Social media handles
* Personal information

Reject or quarantine flagged content.

---

## Community Moderation

Users can:

* Report posts
* Report comments
* Report chats

Rule:

If flags ≥ 5

Automatically:

status = hidden

Pending review.

---

## Shadow Ban System

Moderators can:

* Shadowban anonymous accounts
* Prevent feed visibility
* Prevent chat matching

User should not be notified.
